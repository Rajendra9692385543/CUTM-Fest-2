Hackathon Rules & Guidelines

Important rules and guidelines for the 24-hour coding marathon.

Team Formation

Team Requirements

  • Team size: 2-4 members (no solo participation)
  • Mixed teams from different institutions allowed
  • At least one member should be present at all times
  • Team leader will be the main point of contact
  • No team changes after registration closes
Coding Rules

Development Guidelines

  • Code must be written during the 24-hour period
  • Pre-existing libraries and frameworks allowed
  • No pre-written code specific to the problem
  • All code must be original and team-created
  • Open source tools and APIs are permitted
Submission

What to Submit?

  • Working prototype or demo
  • Source code (GitHub repository)
  • Presentation slides (5-10 minutes)
  • Project documentation
  • Video demo (optional but recommended)
